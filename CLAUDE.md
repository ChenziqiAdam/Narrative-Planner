## 背景说明

该项目的介绍可以阅读[项目介绍.md]，相关参考文献可以阅读[参考文献.md],具体的开发需求可以阅读[prd.md]文件。

## 强制要求

- **在处理具有 MECE 关系的代码文件或逻辑时，充分利用 `Subagent` 或 `Task` 能力，做到并行工作**
- 先写文档，再写代码，以文档指导代码编写，在没有对应的文档时禁止编写代码
- 包管理器必须用 `pnpm`
- 在执行命令之前必须使用 `pwd` 查看当前路径
- 遇到不确定的技术问题时必须使用 `web-search-prime` 和 `web-reader`，必要时使用 `zread` 来查找库说明和样例
- 禁止手动修改 `package.json` 等依赖配置文件，必须用包管理器来处理。例如 `pnpm add` 命令。
- git commit 信息使用约定式提交风格，例如 "feat(frontend): ..."
- 编码规范遵循最佳实践
- 时刻思考“高内聚，低耦合”的编程哲学
- 前端需要按照 `docs/MCP_BROWSER_VISION_WORKFLOW.md` 文件中的步骤来严格执行
