# Planner + Memory 优化工作汇报（通俗版）

日期：2026-04-03

## 一句话总结

这次我们把系统做成了“更容易联调、更容易观察、更容易解释”的状态：  
同样一轮对话，前后端拿到的字段统一了；每次事件提取和合并过程都能看到关键线索；对比页面也能直接看到调试信息，不再靠猜。

---

## 1. 我们做了什么？有什么用？

### 1) 统一了接口返回格式（Baseline 和 Planner）

**做了什么：**
- 把两条链路（baseline / planner）的回复字段统一成一套结构。
- 关键字段包括：`question`、`action`、`done`、`extracted_events`、`graph_update`、`current_graph_state`、`session_metrics`、`debug_trace` 等。

**作用是什么：**
- 前端不用再为两套接口写两套兼容逻辑，开发更省时间。
- 联调时不容易出现“这个接口有、那个接口没有”的问题。
- 后续加功能（比如更多指标）可以一次性接入，成本更低。

---

### 2) 打通了“可观测链路”（提取 -> 合并 -> 返回）

**做了什么：**
- 在事件提取阶段记录调试信息（候选数量、命中线索、异常/回退等）。
- 在合并阶段记录“为什么新增/更新/忽略某个事件”的决策信息。
- 在编排层把这些信息整理成 `debug_trace` 返回给前端和调试页。

**作用是什么：**
- 以前出现“为什么这条事件没进图谱”只能猜；现在可以直接看到原因。
- 排查问题速度更快，减少反复试错。
- 为后续优化 planner / memory 提供了可量化依据，而不是靠感觉。

---

### 3) 对比页面增加了调试展示（小标签 + 详情）

**做了什么：**
- 在消息气泡旁展示关键调试指标（如候选数、合并决策数、回退次数）。
- 支持悬浮查看详细调试数据，方便定位某轮问题。

**作用是什么：**
- 产品同学、算法同学、工程同学可以看同一份证据，不用反复口头解释。
- Demo 时可直接演示“系统为什么这样问、为什么这样更新图谱”。
- 对外展示时可信度更高，不是黑盒效果。

---

### 4) 修复了两个关键测试问题

**做了什么：**
- 提升了职业相关事件识别（补充“纺织厂”等关键词触发）。
- 优化了中等置信度情况下的合并校验逻辑（结合槽位匹配做轻量加分）。

**作用是什么：**
- 事件识别更稳，不容易漏掉明显信息。
- 合并行为更符合预期，减少“该更新却没更新”的情况。
- 回归测试通过，保证改动不是“看起来有效”而是真正稳定。

---

### 5) 完成了端到端 Demo 验证

**做了什么：**
- 跑通了 `planner/start -> planner/reply` 的多轮流程。
- 验证接口会返回统一字段和 `debug_trace`。
- 验证图谱更新字段（`graph_update`、`current_graph_state`）可用。

**作用是什么：**
- 说明这次改动不是停留在代码层，而是实际可跑、可展示、可联调。
- 为后续小组协作（前端看板、评测、策略优化）提供了可用底座。

---

## 2. 这次改动的核心价值（用业务话说）

1. **沟通成本下降**  
大家现在看同一套返回字段和调试信息，讨论效率明显更高。

2. **定位问题更快**  
“出问题在哪一层”从猜测变成可观测，能更快修。

3. **后续优化更可控**  
做 planner 和 memory 优化时，可以直接对比指标和行为变化，不再盲改。

4. **演示效果更完整**  
不只是“能问答”，还可以解释“为什么这么问、为什么这么改图谱”。

---

## 3. 后续建议（你本周可继续推进）

1. 做一个“每轮质量卡片”  
把问题质量、信息增益、覆盖变化做成固定展示，便于周会汇报。

2. 做“失败样例清单”  
把提取失败/合并异常场景沉淀成样例库，后续迭代能持续回归。

3. 做“优化前后对比表”  
同一批访谈样本对比改动前后，直接展示收益（覆盖率、有效事件数、追问质量）。

## 5. 新增改动记录（已标明改动内容）

### 改动 A：低信息增益时自动切题/收尾

**改动内容：**
- 新增“连续低增益检测”规则。
- 连续低增益达到阈值后，系统会从 `continue` 自动转为 `next_phase`（切话题）。
- 当覆盖率已经较高且仍连续低增益时，会建议 `end`（温和收尾）。

**作用：**
- 避免在“老人想不起来/信息很少”的情况下反复追问同一个点。
- 对话更自然，体验更像真人访谈。

---

### 改动 B：切题时避免重复同一句话

**改动内容：**
- 给切题兜底问题加了“轮换问法”。
- 连续切题时不再重复同一句，而是自动换成不同表达。

**作用：**
- 减少机械感。
- 即使模型临时不可用，对话也更流畅。

---

### 改动 C：切题不再随机，优先补低覆盖主题

**改动内容：**
- 系统会优先选择“覆盖率低”的主题去切题，而不是随便换话题。
- 新增推荐字段：`recommended_theme_id`、`recommended_theme_title`。

**作用：**
- 每次切题都更有目标，能更快补齐人生故事覆盖面。
- 提升访谈效率，不浪费轮次。

---

### 改动 D：把策略触发原因写进调试信息

**改动内容：**
- 在每轮 `debug_trace.planning` 中新增可观测字段：
  - `low_info_streak`
  - `prefer_breadth_switch`
  - `suggest_close`
  - `fallback_repeat_count`
  - `recommended_theme_id`
  - `recommended_theme_title`
  - `next_action`

**作用：**
- 可以直接解释“为什么这轮切题/为什么这轮收尾”。
- 方便前端看板展示，也方便组内复盘。

---

### 改动 E：稳定性保障（测试）

**改动内容：**
- 新增并补齐对应单测，覆盖：
  - 低增益触发切题
  - 低增益+高覆盖触发收尾
  - 重复兜底计数
  - 推荐低覆盖主题选择

**作用：**
- 每次优化都有自动化验证，不容易改坏现有能力。
- 当前测试结果：`34 passed`。
